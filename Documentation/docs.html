Interaction Design & User Manual for F1 Steering-Wheel Display

Author: [Popescu Mara, Sarghi Vladut]

Abstract

This document presents the human–computer interaction (HCI) design for the central display on a Formula 1 steering wheel, covering interaction design patterns, rationale behind design decisions, and a concise user manual for drivers.

Interaction Design Patterns

1. Real-Time Dashboard

Displays critical telemetry (speed, gear, lap time) with high-contrast numbers. Uses color coding to indicate safe (white), caution (yellow), and danger (red) states.

2. Mini‑Map Navigation

A top-left track map shows current position and upcoming turn, providing glanceable navigation cues.

3. Weather and track conditions

A page dedicated for the weather which indicates where exactly the rain clouds are, also in the top left indicating the speed level and the direction of the wind and in the center-bottom an AI recomandation for the best tyre for the track conditions

4. Alert & Acknowledgment

Visual warning which pop up when an error appears.

Design Decisions

Color choices

Color choices align with the general color theme of Formula1 for an easier transition to a new display.

Minimalist Layout

Prioritized only essential information to reduce cognitive load. Complex settings (brake bias, engine modes) moved to rotary dials around the display.

Color of the team

Used the pricipal color of the team but keeping in mind the needed for contrast.

User Manual

The following steps guide drivers in interpreting and interacting with the steering‑wheel display:

  1. Startup Self‑Test: After ignition, the display cycles through segments. Confirm that all segments illuminate; if a segment is missing, pit crew intervention is required.
  2. Reading Telemetry:
    • Center: Current speed in km/h.
    • Upper-right: Selected gear.
    • Lower-left: Lap time delta (+/- from best).
  3. Navigating Modes: The driver has 2 different possibilities to navigate beetween the pages either with a little slide or with a button.
